# The Sweepr Handbook

> The single source of truth for how Sweepr works — for the people who build it,
> the cleaners who power it, and the customers who trust it.

This handbook is modeled on the public, version-controlled handbooks of companies
like GitLab and Basecamp. It lives in the repository so it changes the same way
the product does: through pull requests, review, and history. If something here is
wrong, out of date, or unclear, **fix it in a PR** — the handbook is never "done."

## Document conventions

- **MUST / MUST NOT / SHOULD** follow [RFC 2119](https://www.rfc-editor.org/rfc/rfc2119)
  meaning. "MUST" is a hard requirement enforced in code or policy.
- Status badges: 🟢 active · 🟡 in progress · 🔴 required / blocking · ⚪ planned.
- Code, table, and field names are written in `monospace` and match the actual
  schema in `packages/db`.
